# Gap B3: 步态过渡的相位保持混合 (Phase-Preserving Gait Transition Blending)

## 研究综述

本文档记录了 SESSION-049 针对 Gap B3（步态过渡的相位保持混合）的深度研究成果。核心课题是**基于标记的动态时间规整 (Marker-based Dynamic Time Warping, DTW)**，目标是在 walk/run/sneak 之间实现无滑步的平滑过渡。

## 1. 核心参考人物与资料

### 1.1 David Rosen — Wolfire Games (GDC 2014)

**演讲**: _Animation Bootcamp: An Indie Approach to Procedural Animation_ [1]

David Rosen 在 GDC 2014 的演讲中展示了 Overgrowth 游戏的程序化动画系统，核心技术点：

**Stride Wheel（步幅轮）概念**：使用类似测量轮的技术，根据角色在地面上实际移动的距离来推进动画关键帧。这将动画进度直接绑定到角色的物理运动，确保脚步始终与地面同步，角色"永远不会漂浮"。

**Synchronized Blend（同步混合）**：在 walk 和 run 之间的任意中间速度，系统同时混合两种动画的关键帧和各自的步幅大小。不同运动速度通过调整"步幅轮"的大小来适配——走路动画有更小的步幅，因此需要更小的轮。

**Bounce Gravity（弹跳重力）**：角色肢体围绕质心产生弹跳。由于重力是恒定力，弹跳在速度越快时越平坦——高速时每步之间的时间更短，角色下落时间更少，弹跳轨迹更浅。

**关键洞察**：仅用 13 个关键帧就实现了比 Assassin's Creed 数百个动画更流畅的效果，核心在于将物理驱动与相位同步结合。

### 1.2 Unreal Engine Sync Groups & Sync Markers [2]

UE 的 Sync Groups 系统是工业级同步标记实现的标杆：

**Leader-Follower 架构**：Sync Group 中权重最大的动画成为 Leader，其他动画作为 Follower。Leader 提供主时间线长度，Follower 缩放自身时间线以匹配 Leader。当权重翻转时，Follower 自动升级为新 Leader。

**Marker-Based Animation Syncing**：在动画序列中添加同步标记（如 "Left Foot Down"、"Right Foot Down"），系统基于 Leader 在其标记之间的相对位置来同步 Follower。例如，如果 Leader 处于左脚和右脚标记之间的 25% 位置，Follower 也会同步到其对应标记之间的 25% 位置。

**关键规则**：
- 只有同一 Sync Group 中的动画才会同步
- 只有所有动画共有的标记才会被同步
- 如果某个动画缺少某个标记，该标记在所有动画中都会被忽略
- 当动画有足够的匹配标记时，自动使用标记同步；否则退回到缩放长度同步

### 1.3 Bruderlin & Williams — Motion Signal Processing (SIGGRAPH 1995) [3]

开创性论文，首次将信号处理技术应用于动画运动：

**Dynamic Time Warping for Motion Blending**：使用 DTW 在混合前对齐运动信号。核心思想是将运动数据视为时间序列信号，通过 DTW 找到两个不同长度/速度的运动之间的最优对齐路径，然后在对齐后的空间中进行插值混合。

**多目标插值**：支持在多个运动之间进行加权插值，权重可以随时间变化，实现平滑的速度过渡。

### 1.4 Kovar & Gleicher — Registration Curves (SCA 2003) [4]

**Registration Curves**：自动确定输入运动之间的时间、局部坐标系和约束关系。核心贡献是 **Timewarp Curve** 的自动构建——通过约束匹配（如脚部接触事件）自动生成时间规整曲线，无需手动标注。

**约束匹配算法**：自动检测运动中的约束事件（脚部植根、手部接触等），建立跨运动的约束对应关系，然后基于这些对应关系构建时间规整函数。

### 1.5 Ménardais et al. — Synchronization for Dynamic Blending (SCA 2004) [5]

**基于支撑相位的时间规整**：提出了一种高级时间规整算法，使用运动的支撑相位序列（support phases）进行同步。算法还使用代数关系来检测不兼容的运动并选择同步元素。

**核心创新**：将步态周期分解为离散的支撑相位（单脚支撑、双脚支撑、飞行相位），在相位边界处建立对应关系，然后在相位内部进行线性时间规整。

### 1.6 Rune Skovbo Johansen — Locomotion System (2009) [6]

**半程序化动画框架**：自动混合关键帧或动捕的走路和跑步循环，然后调整腿部骨骼运动以适应地形。使用简单的周期对齐来同步 walk 和 run 循环，然后进行混合。

## 2. 核心算法：Marker-based Phase Warping

综合以上研究，Gap B3 的破局算法如下：

### 2.1 Sync Marker 定义

为每个步态循环定义同步标记：
- `0.0` = 左脚落地 (Left Foot Down)
- `0.5` = 右脚落地 (Right Foot Down)

这些标记将步态周期分为两个半周期，每个半周期代表一步。

### 2.2 Phase Warping 算法

给定两个步态 A（如 walk）和 B（如 run），混合权重 α ∈ [0, 1]：

1. **标记对齐**：确定当前 Leader（权重较大的步态）的相位 p_leader
2. **相位映射**：将 p_leader 映射到 Follower 的对应相位 p_follower，确保同步标记对齐
3. **播放速率调整**：计算 Follower 需要的播放速率缩放因子，使其标记与 Leader 同步
4. **骨骼插值**：在对齐后的相位空间中，对骨骼坐标进行加权插值

### 2.3 步幅混合

步幅大小也需要同步混合：
```
stride_blended = stride_walk * (1 - α) + stride_run * α
```

根据混合后的步幅调整根运动速度，确保脚不滑。

### 2.4 弹跳高度自适应

根据 David Rosen 的洞察，弹跳高度与速度成反比：
```
bounce_height = base_bounce * (stride_period / reference_period)
```

速度越快，步幅周期越短，弹跳越平坦。
