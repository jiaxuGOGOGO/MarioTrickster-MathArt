# 🤖 如何与 AI 协作（对话循环指南）

为了让项目实现高质量的不断进化迭代，请在每次新开对话框时，遵循以下**开始 → 过程 → 结束**的良性循环模式。

---

## 🟢 1. 如何开始新对话

每次打开一个新的 Manus 对话框，**第一句话永远是**：

> "这是我的 GitHub 项目 https://github.com/jiaxuGOGOGO/MarioTrickster-MathArt ，请读取 SESSION_HANDOFF.md 继续工作。"

**AI 会做什么：**
1. 克隆你的项目。
2. 读取 `SESSION_HANDOFF.md` 了解当前版本、待办任务和你的核心愿景。
3. 给你弹出一个**选项菜单**，问你接下来想做什么（比如：蒸馏知识、继续开发、诊断停滞等）。

---

## 🟡 2. 过程中的反馈与协作

根据你选择的任务，过程中的协作模式如下：

### 场景 A：你想喂给它新知识（蒸馏）
1. **你**：把 PDF 书籍、参考图片（Sprite）拖进对话框，说："分析这个加入知识库" 或 "把这本书蒸馏进项目"。
2. **AI**：阅读、提取数学和美术规则，写入项目的 `knowledge/` 目录，并优化相关数学模型。
3. **AI**：推送到 GitHub。
4. **你**：在本地电脑运行 `git pull`，然后运行测试确认。

### 场景 B：你想让它继续写代码（开发）
1. **你**：在菜单里选择"继续开发"，或者直接说："开始做 TASK-009"。
2. **AI**：写代码、写测试、运行测试。
3. **AI**：推送到 GitHub。
4. **你**：在本地电脑运行 `git pull`，测试新功能，把终端输出（报错或成功）复制给 AI。

### 场景 C：本地进化卡住了（诊断）
1. **你**：如果本地生成的图片一直一样，或者报错了，把终端日志复制进对话框，说："本地迭代卡住了，帮我诊断"。
2. **AI**：分析日志，找出数学与美术的冲突点，出具诊断报告，并修改代码修复。

---

## 🔴 3. 对话快结束时如何刷新状态

当一个任务做完，或者你准备关掉电脑休息时，**最后一句话永远是**：

> "今天的任务差不多了，请帮我全面审计一下目前项目和我的需求之间还有什么差距，更新待办列表，并刷新 SESSION_HANDOFF.md 和 PROJECT_BRAIN.json 的状态，然后推送到 GitHub。"

**AI 会做什么：**
1. 审视今天做过的所有修改。
2. 对比你的核心愿景，找出还没实现的新差距。
3. 把新差距写进待办列表（TASK-xxx）。
4. 更新版本号、测试数量、最后会话记录。
5. 推送到 GitHub。

这样，当你明天打开一个**全新的对话框**时，重复第 1 步，AI 就能完美无缝地接上今天的进度！
