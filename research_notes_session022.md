# SESSION-022 外部资料关键发现（阶段性）

## 1. 角色程序化像素动画：lookup / UV 映射式 2D 像素动画

来源：<https://blog.derlin.ch/this-guy-may-just-have-revolutionized-2d-pixel-animation/>

### 核心思路

该方案将角色动画与角色外观解耦。动画层不直接存储最终颜色，而是将像素的 RG 通道编码为“在另一张 lookup 贴图中的坐标”。渲染时再由 lookup 贴图决定最终颜色与部件外观。这样，角色的服装、配件、局部造型可以在**不重绘整套动画**的前提下统一替换。

### 对本项目的直接启发

1. `character_renderer.py` 已具备角色部件装配与姿态控制能力，但当前缺少**跨状态统一换装/换色/换局部细节**的机制。可以新增一个“部件 ID / 材质 ID 图层”，在最终着色阶段再映射到调色板与细节纹理。
2. 多状态精灵（idle / walk / jump / attack）若共享同一套 lookup 语义图，可显著降低状态间风格漂移。
3. 这比简单复制逐帧 PNG 更适合项目“持续进化”的目标，因为它把“动画骨架”和“视觉身份”拆成了两个可独立优化的层。

### 拟转化为工程动作

- 在角色渲染阶段输出 `part_id_map` 或 `material_id_map`。
- 在 `pipeline.py` 增加角色图层重着色与换装步骤，而不是只输出最终 RGBA。
- 为未来多状态角色生成预留“统一 lookup / palette contract”。

## 2. Procedural Creature Generation and Animation for Games（PDF 初步核验）

来源：<https://nccastaff.bournemouth.ac.uk/jmacey/MastersProject/MSc22/01/ProceduralCreatureGenerationandAnimationforGames.pdf>

### 当前状态

已确认该 PDF 可访问，内容聚焦“程序化生物生成与动画”。当前仅完成可访问性核验，尚未深入提炼章节细节。

### 已确认价值

1. 该文档明显聚焦“参数化生物形体 + 动画控制”的组合，这与项目当前从几何形状走向角色/生物资产的演进方向高度相关。
2. 可作为后续补充角色骨架、体型参数空间、附属肢体运动策略的候选参考。

### 后续处理建议

- 使用文本抽取方式直接读取 PDF 内容，优先提炼：角色部件参数化、动作控制策略、输出约束。
- 若其方案偏 3D，可仅吸收其“参数空间设计”和“动作控制抽象”，避免把项目带向重型 3D 依赖。

## 当前阶段结论

现有外部资料已经进一步支持以下判断：

1. 项目最大的真实落地缺口仍然是**角色资产管线**，不是几何 SDF 再堆功能。
2. 角色系统下一步不应只做“能渲染一个角色”，而应直接设计为：
   - 可多状态输出；
   - 可局部换装/换色；
   - 可共享统一调色板/材质语义；
   - 可与后续导出和质量评估结合。
3. 这意味着后续优化重点应放在：
   - `character_renderer.py` 与 `pipeline.py` 的实质接线；
   - 角色状态组生成；
   - 可复用的角色语义图层；
   - 面向跨会话迭代的资产记忆与去重机制。
