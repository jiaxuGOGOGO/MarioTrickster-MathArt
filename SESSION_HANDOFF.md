### Layer 1: Internal Evaluation

The repository now has a dedicated **Unity-native runtime evaluation path**. `UnityURP2DEvolutionBridge.evaluate_native_stack()` verifies that the importer generator exists, the Secondary Texture postprocessor exists, VAT playback/runtime files are emitted, VAT manifests are structurally valid, and the Taichi backend can produce finite cloth-cache frames for downstream Unity replay.

`knowledge/unity_urp_2d_rules.md` now persists the engineering conclusions distilled from Unity URP 2D documentation, Dead Cells-style multi-channel 2D lighting practice, XPBD cloth stability, and VAT replay strategy. The repository therefore remembers not just *what* was implemented, but *why this architecture was selected*.

The new `.unity_urp_2d_state.json` persists cycles, pass streaks, VAT size trends, and native bridge health. More importantly, `EvolutionOrchestrator.run_full_cycle()` now includes a **Unified Bridge Suite** that executes:
