## 当前可确认结论

当前问题并不应通过“关闭阻尼”或“魔法补偿重力”来规避，而应从积分路径上确保**外力场重力**与**内部约束阻尼**在数学上解耦。修复方向应优先审查求解器是否把 Rayleigh damping 直接乘在了全局绝对速度上；若是，则这会错误吞噬纯自由落体中本应完整保留的重力速度增量。

同时，若求解器声称采用半隐式欧拉，则需要验证重力是否按 `v_{n+1} = v_n + g·dt`、`x_{n+1} = x_n + v_{n+1}·dt` 的顺序进入积分。只要系统不存在约束相对运动，阻尼就不应改变这一自由落体基线。后续将继续从论文正文与项目现有代码中提炼精确代数落点，并把这些约束转化为严格测试断言。

## 追加研究结论

### Box2D / Erin Catto 工业纪律

Box2D 官方文档明确把 damping 定义为“用于降低刚体世界速度”的机制，并给出一阶速度阻尼方程 `dv/dt + c v = 0` 及其离散近似 `v(t+h) ≈ v(t) / (1 + c h)`。这说明**全局线性阻尼**本质上是对刚体绝对速度的经验衰减，而不是对内部约束相对速度的物理投影。因此，在 XPBD 软体/弹簧系统里，如果把此类全局速度衰减误套到“仅受重力外场驱动、没有内部相对形变”的自由落体场景，就会直接吞噬重力积分得到的速度增量，破坏解析基线。

### NASA 建模与仿真验证纪律

NASA-STD-7009B 的标题即为《Standard for Models and Simulations》，其作用不是鼓励“看起来合理”的主观验证，而是要求模型与仿真具备明确、可追溯、可量化的验证与确认证据。对应到本项目，自由落体不应仅凭视觉轨迹判断“像抛物线”，而应以解析解 `s = 1/2·g·t^2` 为金标准，直接使用严格浮点容差完成自动断言，形成可回归、可审计、可复现实证。

### 当前对代码改造的直接含义

综合 XPBD 原论文与 Box2D 工业文档，本轮修复的正确方向应当是：

1. 保留重力等外力场在半隐式欧拉下的完整积分；
2. 仅让 XPBD 的耗散项作用于约束引起的相对运动，或至少避免其在无约束自由落体阶段衰减绝对速度；
3. 测试层面必须将自由落体解析解升级为高精度基线，而非继续容忍因错误阻尼耦合造成的系统性偏差。

### 可直接指导实现的原始依据摘录

> XPBD 2016 将额外阻尼明确建模为 Rayleigh dissipation potential：`D(x, v) = 1/2 · Cdot(x)^T β Cdot(x) = 1/2 · v^T ∇C^T β ∇C v`，并给出 `f_damp = -∇C^T β ∇C v`。这说明阻尼力是通过**约束梯度 `∇C` 投影**得到的，天然绑定于约束相对运动，而不是对整体绝对平移进行无差别削减。

> XPBD 2016 的预测步写为 `x̃ ← x^n + Δt v^n + Δt^2 M^{-1} f_ext(x^n)`，随后更新 `v^{n+1} ← (x^{n+1} - x^n) / Δt`。这意味着外力场应先无损进入预测/积分，而不是先被全局阻尼吞掉。

> Box2D 官方文档把 damping 直接定义为“用于降低刚体世界速度”，并给出 `dv/dt + c v = 0` 的经验更新。该机制适用于刚体世界速度管理，但若原样挪到 XPBD 软体自由落体，就会与“约束诱导耗散”混线。

> Müller 等 2020 在刚体 XPBD 中把关节阻尼放在独立的 velocity pass 内，并采用 `Δv ← (v2 - v1) min(μ_lin h, 1)` 与 `Δω ← (ω2 - ω1) min(μ_ang h, 1)`。这再次强调阻尼应作用在**相对速度差**上，而不是对系统共享的外场平移速度统一打折。

> NASA-STD-7009B 围绕模型与仿真要求维护 verification / validation domain、记录不确定性并提供客观证据；标准还明确提到 code verification 与 solution verification 以及对数值误差对精度影响的评估。对应到本项目，自由落体精度必须用解析基线做 solution verification，而不是主观观感。

### 本轮实现最高准则

| 准则 | 实现含义 |
|---|---|
| **外力—阻尼解耦** | 重力必须完整进入半隐式欧拉预测步；阻尼不得在无约束自由落体中衰减绝对速度。 |
| **阻尼只看相对运动** | 若保留 Rayleigh/速度阻尼，应投影到约束相对速度或内部形变速度，而不是全局刚体平移。 |
| **2D/3D 双轨一致** | `xpbd_solver.py` 与 `xpbd_solver_3d.py` 需要同步修复，避免维度行为分叉。 |
| **解析基线优先** | 自由落体测试必须直接对 `s = 1/2 g t^2` 做严格断言，目标至少 `rtol=1e-4`。 |
| **零回归守卫** | 需同时验证弹簧柔度、布料稳定性、CCD 与 RL 参考动作相关高级测试不回退。 |
