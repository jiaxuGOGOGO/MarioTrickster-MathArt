# 蒸馏知识接口规范

> 本文件定义了如何向 MarioTrickster-MathArt 项目注入新的蒸馏知识。

## 工作流

```
你读书 → 和 AI 对话提炼规则 → 追加到 knowledge/*.md → git push → CI 自动验证
```

## knowledge/ 目录结构

| 文件 | 内容 | 影响的代码模块 |
|------|------|---------------|
| `anatomy.md` | 关节可动域、头身比、肌肉联动 | `animation/skeleton.py` 的 ROM 约束 |
| `perspective.md` | 透视法则、深度暗示 | 未来 3D→2D 降维模块 |
| `color_light.md` | 暖光冷影、色阶系统、限色约束 | `oklab/palette.py` 的和谐规则 |
| `animation.md` | 12 法则数学映射、帧数约定 | `animation/presets.py` 的预设参数 |
| `unity_rules.md` | PPU/Filter/Pivot/命名/目录 | `export/bridge.py` 的校验规则 |

## 如何添加新知识

### 1. 追加到已有文件

如果新知识属于上述某个类别，直接在对应 `.md` 文件末尾追加新章节：

```markdown
## 新章节标题（来源：书名/作者）

具体规则描述...

| 参数 | 值 | 说明 |
|------|-----|------|
| ... | ... | ... |
```

### 2. 创建新知识文件

如果新知识不属于任何已有类别（比如"粒子系统"、"地形生成"）：

1. 在 `knowledge/` 下创建新 `.md` 文件
2. 文件开头写明来源
3. 用表格组织可参数化的规则
4. 在本文件的目录结构表中追加一行

### 3. 知识 → 代码的转化

蒸馏知识不会自动变成代码。转化路径：

- **硬约束**（如关节角度范围）→ 直接写入代码常量
- **软规则**（如"暖光冷影"）→ 作为默认参数，允许覆盖
- **启发式**（如"重叠产生深度"）→ 作为文档注释，指导手动调参

### 4. 蒸馏质量检查

好的蒸馏知识应该是：
- **可参数化**：能转化为数字、范围、枚举值
- **可验证**：能写成测试断言
- **有来源**：标注书名/作者/页码
- **无歧义**：不同 AI 读到同样的描述应产出相同的代码
